### Быстрый старт (разработка)

**Требования:** Docker Desktop, [.NET SDK 10](https://dotnet.microsoft.com/download), Node.js ≥ 20.19 (см. [`.nvmrc`](.nvmrc)).

```bash
# 1. Зависимости JS (workspaces: Web, Mobile, client-core)
npm install

# 2. PostgreSQL + миграции (Windows PowerShell)
./Scripts/setup-local-postgres.ps1

# 3. API (JWT в Development генерируется автоматически)
dotnet run --project Flora.API
# → http://localhost:5284

# 4. Web
cp Apps/Web/.env.example Apps/Web/.env.local   # Linux/macOS
# copy Apps\Web\.env.example Apps\Web\.env.local   # Windows
cd Apps/Web && npm run dev
# → http://localhost:3000
```

**Android (USB):** VS Code task **Flora Android: debug (USB)** — PostgreSQL, API, установка Flora Dev, Metro одним кликом. CLI: `./Scripts/mobile-debug-android.ps1`.

На Windows удобнее использовать готовые задачи из [`.vscode/tasks.json`](.vscode/tasks.json): **Flora: API + Web dev localhost**, **Flora Android: debug (USB)**.

### Некоммерческий фундамент
Для проекта, который стремится стать всеобъемлющим инструментом для всех людей в новом веке информационных технологий **недопустима** коммерческая пытка пользователей. Мы исключаем из проекта абсолютно всю деструктивную коммерцию, оставляя право лишь на точечную экосистеме **LUNA**, которая покроет собой узкий круг задач для бизнеса, которые никак не будут касаться незаинтересованного пользователя. 

В социальной сети **отсутствует реклама** и платное продвижение, а бизнес проекты и бренды отделены стеной от тех пользователей, которые не хотят чтобы их касался этот шум. Вы не увидите таргетированной рекламы, а посты брендов продвигаются ровно также, как и посты обычных людей, но брендированный контент обязательно помечается и отслеживается, а любой пользователь может выключить его отображение. Также с социальной сети **отсутствует платная кастомизация**, которая разделяет пользователей на бессмысленные слои.

### Прозрачность и открытый исходный код
Мы считаем, что пользователь имеет право знать, как работает инструмент, и что он не работает против него. А любые **разработчики**, которые хотят участвовать в проекте, должны иметь эту возможность. Прозрачность является **необходимым правилом**, чтобы выстроить доверие и уверенно развивать проект.

### Безопасность и шифрование
Любой цифровой инструмент, которым пользуется человек, должен быть для него безопасным. Мы уделяем **огромное внимание** к безопасности пользователей.

**FSCP** (Flora Secure Chat Protocol)  —  собственный протокол **E2E** шифрования, который гарантирует что ваши сообщения не будут прочитаны третьими лицами, а пользовательский опыт от обмена сообщениями не будет ухудшен из-за сложностей в синхронизации. Ваш ключ привязан к вашим учетным данным и легко передается от устройства к устройству без лишних действий. Спецификация: [`docs/fscp/FSCP.md`](docs/fscp/FSCP.md).

Философская сторона вопроса о сквозном шифровании не столь сложна, как это пытаются преподнести людям. Все **аргументы против** E2E обычно сводятся к трем манипуляциям, которые легко разбиваются о реальность:

* **«Шифрование защищает преступность»** –  Злоумышленники всегда найдут способ обойти ограничения, создав собственные инструменты. Отмена сквозного шифрования не остановит криминал  –  она лишь лишит защиты миллиарды обычных людей, сделав их уязвимыми перед **хакерами**, мошенниками и **утечками**, а также перед индивидуальными угрозами личной жизни или даже домашнего насилия. 
* **«Честному человеку нечего скрывать»** –  Опасное заблуждение. Приватность  –  это не признак вины, а базовое право на личные границы. Без права на тайну переписки реальная свобода выбора и слова просто невозможна. Честному человеку **всегда есть** что скрывать от третьих лиц.
* **«Контроль равен безопасности»** –  В действительности тотальный надзор создает куда большую угрозу  –  риск системного злоупотребления контролем и превращения цифровой среды в инструмент давления. Те самые **честные люди** которым нечего скрывать оказываются **преступниками** по мнению репрессивного режима, защищая свои базовые и неотъемлемые человеческие права.

**Истинная цифровая безопасность** –  это не когда за пользователем следят ради его же блага. Это когда никто, включая создателей платформы, физически не имеет технической возможности вторгнуться в его личное пространство. Именно эту фундаментальную свободу и защищает **FSCP**. Всем сторонникам репрессий рекомендую к прочтению **1984**.

### Дизайн и кастомизация
Мы стремимся к тому идеалу в науке дизайна, который сможет дать пользователю высочайший уровень эргономики и эстетического удовольствия от пользования нашими сервисами. Мы также стремимся дать возможность пользователю настроить свое пространство так, чтобы оно было ему комфортно. Не только по науке, но и на любой вкус и цвет.

### Индивидуальный пользовательский опыт и рекомендации
Для того, чтобы пользователь получал самую интересную для него информацию и медиа, мы создали **FIRA**.

**FIRA** (Flora Individual Recommendation Algorithm)  –  алгоритм, цель которого проста: дать пользователю **максимально эффективные** личные рекомендации. Спецификация: [`docs/fira/FIRA.md`](docs/fira/FIRA.md). FIRA подразделяется на модули на основе своего ядра:
* **FIRA-F**  –  **лента**
* **FIRA-M**  –  **музыка**
* **FIRA-P**  –  **люди**
* **FIRA-C**  –  **сообщества**

В будущем алгоритм будет расширяться и на другие независимые модули.
